# Raw-источники

`knowledge/raw/` хранит структуру и описания инженерных источников. Сами тяжёлые Excel, PDF, DWG, DXF, ZIP и конфиденциальные материалы по умолчанию не коммитятся.

## Excel-файлы

Excel-файлы используются как расчётный oracle. Они нужны для сверки результатов, восстановления логики расчёта, проверки массы, профилей, шагов, нагрузок, создания acceptance tests и объяснения расхождений между кодом и эталоном.

Исходные Excel-файлы не изменять. Если нужна рабочая копия, создавать копию с суффиксом `_working`. Выводы из Excel переносить в `knowledge/wiki/parity/`, а расхождения фиксировать в `knowledge/wiki/parity/known-differences.md`.

## Проекты КМ

КМ-проекты используются как реальные инженерные примеры. Они помогают проверять практичность решений, понимать типовые пролёты, профили, узлы и спецификации, а также сравнивать результат калькулятора с инженерной практикой.

КМ-проекты не являются автоматическим oracle для формул. Конфиденциальные проекты не коммитить; использовать только обезличенные выдержки, скриншоты и заметки.

## Нормативные документы

Нормативные документы используются для проверки методик. Если документ нельзя коммитить, хранить только ссылку или описание. В wiki фиксировать краткие выводы и ссылки на разделы; формулы и коэффициенты не выдумывать.

## Защита от случайного коммита

Внутри `knowledge/raw/` действует отдельный `.gitignore`. Он блокирует Excel, PDF, DWG, DXF, RVT, IFC и архивы. В Git должны попадать только:

- README-файлы;
- `.gitkeep`;
- `source-register.md`;
- `.gitignore`;
- обезличенные заметки и выводы, если они добавлены отдельно в wiki.

Перед коммитом всегда проверять `git status`. Если в списке файлов появились `.xlsx`, `.pdf`, `.dwg`, `.dxf`, `.zip` или похожие raw-исходники, их нужно убрать из индекса и проверить путь хранения.
