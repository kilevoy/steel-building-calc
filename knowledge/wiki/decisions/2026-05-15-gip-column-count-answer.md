# Решение ГИПа — подсчёт торцевых колонн

Дата: 2026-05-15.

Статус: `accepted`.

Связанный вопрос: `knowledge/wiki/decisions/pending-gip-column-count-question.md`.

Связанные сценарии: `SCN-BUILDING-COUNT-001`, `SCN-BUILDING-COUNT-002`, `SCN-BUILDING-COUNT-003`.

## Ответ ГИПа

Получено инженерное уточнение:

```text
торцевые колонны считать отдельной группой фахверка/торцевых элементов.
так правильно и так дешевле.

но если здание с краном (без разницы подвесным или опорным),
тогда торцевые колонны считать основными колоннами рам.
```

## Принятое правило

Для обычного здания без крана:

- торцевые колонны не включать в основные группы `edge/middle`;
- торцевые колонны учитывать отдельной группой `endFrame/fachwerk`;
- текущая модель приложения, где `edge/middle` считаются по `interiorFrameCount`, соответствует этому решению.

Для здания с краном:

- торцевые колонны считать основными колоннами рам;
- основные `edge/middle` должны включать торцевые рамные оси;
- не важно, кран подвесной или опорный: сам факт наличия крана переводит торцевые колонны в группу основных рамных колонн.

## Следствие для будущего кода

В будущей модели "Единого здания" подсчёт колонн должен зависеть от признака наличия крана:

```text
если кранов нет:
  main columns = interior frame columns
  end columns = separate endFrame/fachwerk group

если кран есть:
  main columns = all frame columns, including end frames
  end columns are not double-counted as fachwerk
```

Нужно не допустить двойного счёта: если торцевая колонна при кране перешла в основные колонны рам, она не должна второй раз попадать в фахверковые стойки.

## Что это значит для текущего приложения

На момент фиксации решения диагностический блок Summary показывает оба числа: по внутренним рамам и по всем рамам. Это полезно оставить до реализации явного переключения по наличию крана.

Расчётные формулы колонн менять не требуется. Решение касается классификации и количества элементов здания, а не проверки сечения.

## Что нужно сделать следующим шагом

Статус реализации: базовый режим `hasCrane` добавлен в `src/building/unifiedLayout.ts` и Summary. Признак крана берётся из полей "Кран опорный" и "Кран подвесной" на вкладке "Колонна". Подкрановая балка не рассчитывается, если кран не включён.

1. Добавить в `deriveUnifiedBuildingLayout` явный режим подсчёта по наличию крана.
2. Обновить `SCN-BUILDING-COUNT-001...003`: часть сценариев должна проверять здание без крана, часть — с краном.
3. В Summary показать понятную подпись: "без крана — торцы отдельно", "с краном — торцы как основные рамы".
4. Перед включением в массу/стоимость проверить, что фахверк не дублирует основные торцевые колонны.

## Что не менять без отдельного задания

- Не менять расчётные формулы колонн.
- Не менять подбор профилей.
- Не менять generated-файлы.
- Не менять справочники профилей.
- Не пересчитывать текущие acceptance-сценарии без явного обновления модели количества.
