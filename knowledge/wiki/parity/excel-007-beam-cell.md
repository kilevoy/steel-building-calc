# EXCEL-007 — подбор элементов балочной клетки

Источник: `EXCEL-007` из `knowledge/raw/source-register.md`.

Файл: `Подбор элементов балочной клетки v3.1.xlsx`.

Статус ingest: описана структура workbook и основные наблюдаемые входы/выходы. Формулы в код не переносились.

## Назначение

Workbook используется как Excel-oracle candidate для модуля балочной клетки / балки покрытия. Источник важен для сверки входов геометрии, второстепенных балок, главных балок, стоек/колонн, массы и стоимости.

## Листы workbook

| Лист | Назначение по наблюдаемой структуре | Размер | Формулы |
|---|---|---:|---:|
| `Исходные данные` | основной ввод и итоговые подобранные элементы | `A1:X42` | 28 |
| `Расчет ВБ` | расчёт второстепенных балок | `A1:AEU178` | 107337 |
| `Расчет ГБ` | расчёт главных балок | `A1:AG156` | 1227 |
| `Подбор колонн` | подбор стоек/колонн балочной клетки | `A1:AM421` | 4170 |

## Наблюдаемые входы

На листе `Исходные данные` видны:

- габариты балочной клетки;
- размер вдоль ГБ;
- размер поперёк ГБ;
- высота К;
- габариты ячейки;
- пролёт ГБ;
- шаг ГБ;
- конструкция перекрытия;
- нагрузка на перекрытие;
- вид конструкции;
- стоимости швеллера, трубы и двутавра для С245/С345;
- максимальный шаг ВБ.

В справочных вариантах видны конструкции перекрытия: сборные ЖБ плиты, монолитный ЖБ 200 мм, монолитный ЖБ 150 мм, OSB и балка покрытия.

## Наблюдаемые выходы

На листе `Исходные данные` видны итоговые блоки:

- подобранные сечения ВБ;
- общий вес ВБ;
- шаг ВБ;
- профиль ВБ;
- стоимость ВБ;
- подобранные сечения ГБ;
- общий вес ГБ;
- профиль ГБ;
- стоимость ГБ;
- подобранные сечения К;
- вес одной К;
- профиль К;
- итоговая масса без колонн;
- итоговая масса с колоннами.

## Наблюдаемые расчётные блоки

`Расчет ВБ` содержит большой расчёт второстепенных балок и справочные данные профилей. В структурном срезе видны нормальные двутавры, швеллеры, профильные характеристики, шаг ВБ, стоимость, общий вес и профиль ВБ.

`Расчет ГБ` содержит отдельный расчёт главных балок.

`Подбор колонн` содержит подбор стоек/колонн балочной клетки; это не нужно смешивать с основным модулем колонн без отдельной сверки.

## Связь с текущим модулем

Связанная страница модуля: `knowledge/wiki/modules/beam-cell.md`.

Текущий UI использует термин Beam Cell / балка покрытия. При будущей сверке нужно явно сопоставить эти термины с `ВБ`, `ГБ` и `К` из workbook.

## Что требует проверки

- Сопоставить текущие входы модуля с листом `Исходные данные`.
- Уточнить, когда модуль проекта должен считать только балку покрытия, а когда полноценную балочную клетку.
- Сверить критерии подбора ВБ и ГБ.
- Проверить, нужен ли отдельный вывод стоек/колонн `К` в текущем UI.
- Выбрать минимальный acceptance scenario: входы, профиль ВБ, профиль ГБ, масса и стоимость.

## Правила дальнейшего использования

Не изменять Excel-файл. Не переносить формулы в код без отдельного задания и инженерной проверки.

Если создаётся тест-кейс, фиксировать отдельно входы, выходные профили, массы, стоимость и допустимое расхождение.
