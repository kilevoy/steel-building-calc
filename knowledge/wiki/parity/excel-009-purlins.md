# EXCEL-009 — общий калькулятор прогонов

Источник: `EXCEL-009` из `knowledge/raw/source-register.md`.

Файл: `Общий калькулятор прогонов v2.0.xlsx`.

Статус ingest: описана структура workbook и основные наблюдаемые входы/выходы. Формулы в код не переносились.

## Назначение

Workbook используется как Excel-oracle candidate для модуля прогонов. Источник важен для сверки горячекатаных/трубных/швеллерных прогонов, ЛСТК МП350/МП390, шагов, масс, стоимости и ограничений по нагрузкам.

## Важные примечания workbook

На `Лист1` видны текстовые примечания:

- коэффициенты использования "по умолчанию" согласованы с главным конструктором;
- несущая способность посчитана в MicroFE 2015;
- расчётная модель указана как шарнирно опёртая балка пролётом 6 м;
- коэффициент условия работы для `ПZ` и для `ПС/ТПС`;
- раскрепления для профилей приняты с шагом 0,4 м через волну профлиста или сэндвич-панели.

Эти утверждения зафиксированы как наблюдаемые подписи workbook. Их применимость к текущему коду требует отдельной инженерной проверки.

## Axial wind load

В текущем UI проекта есть предупреждение, что осевая ветровая нагрузка для прокатных прогонов пока задана временным допущением.

Структурный ingest `EXCEL-009` не является достаточным подтверждением методики axial wind load. Для снятия предупреждения нужен отдельный разбор формул и acceptance-сценарий.

## Связь с текущим модулем

Связанная страница модуля: `knowledge/wiki/modules/purlins.md`.

При будущей сверке нужно отдельно сравнивать горячекатаные/прокатные варианты и ЛСТК МП350/МП390, потому что workbook содержит разные большие расчётные блоки.

## Что требует проверки

- Сопоставить входы `Лист1` с текущим UI модуля прогонов.
- Проверить методику axial wind load для прокатных прогонов.
- Проверить сценарии со снеговым мешком.
- Проверить установку тяжей и шаг распорок.
- Проверить различие МП350 и МП390.
- Определить минимальный набор acceptance scenarios, чтобы не сравнивать весь workbook.
- Проверить внешние ссылки на `[1]СП 20` и `[1]СП РК EN`, если они участвуют в формулах.

## Правила дальнейшего использования

Не изменять Excel-файл. Из-за большого количества формул читать workbook выборочно и фиксировать только необходимые обезличенные сценарии.

Не переносить коэффициенты и формулы в код без отдельного задания, источника и проверки.
