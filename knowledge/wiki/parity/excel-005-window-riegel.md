# EXCEL-005 — подбор оконных ригелей

Источник: `EXCEL-005` из `knowledge/raw/source-register.md`.

Файл: `Подбор оконых ригелей v2.0.xlsx`.

Статус ingest: описана структура workbook и основные наблюдаемые входы/выходы. Формулы в код не переносились.

## Назначение

Workbook используется как Excel-oracle candidate для модуля оконных ригелей. Его можно применять для сверки структуры входных данных, состава проверок и ожидаемых результатов подбора.

## Листы workbook

| Лист | Назначение по наблюдаемой структуре | Размер | Формулы |
|---|---|---:|---:|
| `Лист1` | основной ввод, выбор параметров и вывод подобранных сечений | `A1:Q53` | 70 |
| `Расчет` | основной расчёт и таблица профилей/кандидатов | `A1:AX413` | 3352 |
| `Лист2` | справочная таблица коэффициентов для устойчивости | `A1:AT26` | 0 |
| `Ветер СП` | расчёт ветровых нагрузок по СП 20.13330.20ХХ | `A1:AM74` | 231 |
| `Ветер по СП РК EN` | расчёт ветровых нагрузок по СП РК EN | `A1:AG48` | 87 |
| `типы окон` | справочник типов окон | требует отдельного просмотра | требует отдельного просмотра |

## Наблюдаемые входы на `Лист1`

- город;
- уровень ответственности;
- высота окна;
- шаг рам;
- тип окна;
- высота здания;
- пролёт здания;
- длина здания;
- тип местности;
- ветровая нагрузка;
- конструкция окна / тип стеклопакета;
- максимальный коэффициент использования.

Также на листе видны переключатели/варианты: тип местности `А/В/С`, ветровые районы, старый/новый подход к ветру, СП РК EN / СП 20.13330.20ХХ, односкатная/двускатная схема.

## Наблюдаемые выходы на `Лист1`

Workbook выводит два блока подбора:

- подобранное сечение нижнего ригеля для типов 1-5 и верхнего ригеля для типов 2-5;
- подобранное сечение верхнего ригеля для типа 1.

Для результатов видны поля: номер, сечение, сталь, вес одного ригеля. В cached-значениях примера встречаются квадратные и прямоугольные трубы, включая варианты с пометкой `90°`.

## Наблюдаемые проверки

На основном листе явно указано, что подбор осуществляется по условиям:

- ограничения предельной гибкости `200`;
- предельного прогиба по вертикали `1/300`;
- предельного прогиба по горизонтали `1/200` пролёта;
- расчёта на прочность.

На листе `Расчет` видны диагностические колонки:

- коэффициент использования по гибкости;
- коэффициент использования по прочности для нижнего ригеля;
- коэффициент использования по прочности для верхнего ригеля;
- коэффициент использования по прогибам для нижнего ригеля;
- коэффициент использования по прогибам для верхнего ригеля;
- признаки прохождения для нижнего и верхнего ригеля.

## Связь с текущим модулем

Связанная страница модуля: `knowledge/wiki/modules/window-riegel.md`.

Текущий код модуля использует generated workbook-логику. При будущей сверке нужно сравнивать не только итоговое сечение, но и диагностические коэффициенты: гибкость, прочность, вертикальный прогиб и горизонтальный прогиб.

## Что требует проверки

- Точная интерпретация всех входных ячеек и переключателей.
- Соответствие текущего UI всем входам Excel.
- Соответствие текущего engine листу `Расчет`.
- Правила выбора между СП 20.13330.20ХХ и СП РК EN.
- Обработка внешних ссылок вида `[1]СП 20` и `[1]СП РК EN`, если они есть в формулах.
- Назначение листа `типы окон` и его связь с коэффициентами приведения момента.

## Правила дальнейшего использования

Не изменять Excel-файл. Если нужен расчётный сценарий, создать отдельное обезличенное описание входов и ожидаемых выходов в wiki/parity или тестовых данных.

Не переносить формулы в код без отдельного задания и инженерной проверки. При расхождении с кодом фиксировать проблему в `known-differences.md`.
