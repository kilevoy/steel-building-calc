# Acceptance scenarios

Эта папка предназначена для маленьких проверяемых сценариев, извлечённых из Excel-oracle, КМ-проектов или внешних репозиториев.

Сценарий должен быть компактным и обезличенным. Нельзя коммитить исходные Excel/PDF/DWG. В сценарий можно переносить только входные значения, ожидаемые результаты, допуск и ссылку на ID источника.

## Шаблон сценария

Каждый сценарий должен содержать:

- ID сценария;
- источник;
- назначение;
- входные данные;
- ожидаемые выходы;
- допуск;
- что не проверяется;
- открытые вопросы.

## Статусы

- `draft` — структура создана, данных недостаточно.
- `candidate` — есть входы и ожидаемые выходы, нужна ручная проверка.
- `accepted` — сценарий принят пользователем или инженерно подтверждён.
- `blocked` — сценарий нельзя использовать из-за расхождения, неполных данных или конфиденциальности.
