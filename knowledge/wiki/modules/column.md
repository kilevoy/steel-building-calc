# Column

Модуль колонны рассчитывает подбор/проверку колонн по входным параметрам здания и нагрузкам.

Известно: default input вынесен в `src/defaults/columnInput.ts`, а smoke-тест запускает `runCalculation(DEFAULT_COLUMN_INPUT)` и проверяет непустой результат.

Модуль связан с нагрузками от здания и с допущением `H = 0`. При изменениях нужно сверять результаты, тесты и предупреждения в UI. Формулы не менять без отдельного инженерного решения.

Связанный Excel-oracle candidate: `EXCEL-010`, описан в `knowledge/wiki/parity/excel-010-column.md`.

При будущей сверке нужно проверять входы здания, тип колонны, нагрузки `N/M`, итоговый профиль, массу и диагностические коэффициенты: прочность, устойчивость и гибкость.
# Длины и высоты колонн

Текущее поведение закреплено тестами в `src/building/layout.test.ts`.

Что принято в коде сейчас:

- для двускатной кровли высота колонны определяется симметрично относительно середины пролёта;
- крайние колонны на отметках `x = 0` и `x = span` имеют базовую высоту здания;
- средняя колонна многопролётной схемы при `x = span / 2` получает добавку от уклона кровли;
- для односкатной кровли высота линейно растёт от низкого края к высокому;
- суммарная длина группы колонн считается как сумма высот позиций, умноженная на количество соответствующих рам/торцов.

Важно: этот блок фиксирует существующее поведение, а не вводит новую расчётную методику. Если ГИП уточнит другую трактовку высот для фахверка, крайних или средних колонн, сначала нужно добавить отдельное решение в `knowledge/wiki/decisions/`, затем менять код и acceptance-сценарии.
