# Инженерная память проекта

`knowledge/` хранит инженерный контекст проекта `steel-building-calc`: структуру модулей, принятые решения, временные допущения, связь с Excel/VELICAN-оракулами и правила работы для будущих AI-агентов.

Эта папка не заменяет исходный код и тесты. Код остаётся источником реализации, тесты — автоматической проверкой поведения, а `knowledge/` — источником понимания: почему решения были приняты, что уже проверено и что требует отдельной инженерной верификации.

`knowledge/raw/` предназначена для описания исходных материалов: Excel, КМ-проектов, нормативных документов и скриншотов. Тяжёлые и конфиденциальные файлы по умолчанию не коммитятся.

## Работа через Google Drive

Проект можно держать в синхронизируемой папке Google Drive, если нужно работать из дома и с работы. При этом Google Drive не заменяет GitHub:

- Google Drive используется как удобная общая рабочая папка;
- GitHub остаётся источником истории версий, коммитов и deploy;
- `knowledge/raw/` может хранить локальные Excel/PDF/КМ-исходники;
- тяжёлые raw-исходники не должны попадать в публичный GitHub.

Не рекомендуется синхронизировать как ценные данные:

- `node_modules/`;
- `dist/`;
- `tmp/`;
- `.env` и другие локальные секреты;
- `*.log`;
- служебные `desktop.ini`.

Рекомендуемый порядок работы:

1. На одном компьютере закончить изменения.
2. Выполнить `git status`.
3. Закоммитить и запушить изменения.
4. Дождаться синхронизации Google Drive.
5. На другом компьютере перед началом выполнить `git status` и `git pull`.

Не работать одновременно с одной и той же Drive-папкой на двух компьютерах.
