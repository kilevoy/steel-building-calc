# Правила для AI-агентов

## Базовые запреты

Не изменять расчётные формулы без отдельного явного задания пользователя. Если формулу всё же нужно изменить, сначала описать причину, риск, ожидаемую проверку и источник подтверждения.

Не менять коэффициенты СП, справочники профилей, generated-файлы, lazy loading климатических данных и GitHub Pages deployment без отдельной задачи.

## Источники и документация

Все инженерные утверждения фиксировать в `knowledge/wiki/`. Все временные допущения заносить в `knowledge/wiki/open-assumptions.md`. Все важные изменения фиксировать в `knowledge/log.md`. Решения пользователя заносить в `knowledge/wiki/decisions/`.

Исходные файлы в `knowledge/raw/` считать источниками истины и не изменять без отдельного разрешения. Excel, PDF, DWG, DXF, ZIP и другие тяжёлые исходники по умолчанию не коммитить. Если источник конфиденциальный, в Git хранить только обезличенное описание и выводы.

Если агент использует данные из Excel или КМ-проекта, он должен указать ID источника из `knowledge/raw/source-register.md`.

## Противоречия

Если есть противоречие между кодом, Excel-оракулом, КМ-проектом, тестом и документацией, не скрывать его. Записать проблему в `knowledge/lint.md` или `knowledge/wiki/parity/known-differences.md`.

## Перед изменением модулей

Перед изменением расчётного модуля читать соответствующую страницу:

- `knowledge/wiki/modules/column.md`
- `knowledge/wiki/modules/truss.md`
- `knowledge/wiki/modules/purlins.md`
- `knowledge/wiki/modules/beam-cell.md`
- `knowledge/wiki/modules/window-riegel.md`
- `knowledge/wiki/modules/crane-beam.md`

После изменения расчётного модуля обновить страницу модуля, `knowledge/wiki/parity/acceptance-tests.md`, `knowledge/wiki/open-assumptions.md` при появлении новых допущений и `knowledge/log.md`.

## Проверки перед коммитом

Перед коммитом запускать `npm run typecheck`, если есть, затем `npm run lint`, `npm test` и `npm run build`. Если скрипт отсутствует, не добавлять его автоматически, а указать это в отчёте.

Не ухудшать текущую структуру lazy loading и bundle size. Не возвращать climate JSON в стартовый bundle. Не удалять предупреждения из UI без инженерной проверки и не скрывать временные допущения от пользователя.
