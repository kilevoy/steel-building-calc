# Lint инженерной памяти

## Противоречия

Активных задокументированных противоречий между wiki и кодом пока нет. Требуется повторная проверка после наполнения `knowledge/`.

## Устаревшие сведения

Проверять после каждого крупного refactor, особенно страницы модулей и checkpoints.

## Страницы без источников

Большинство стартовых страниц описывают текущее понимание проекта и требуют привязки к источникам по мере добавления Excel/КМ/нормативных материалов.

- Проверить методику расчёта горизонтальной реакции `H`.
- Проверить временный axial wind load warning в purlins.
- Убедиться, что `source-register.md` заполняется по мере добавления исходников.
- Проверить, что raw-исходники не попадают в Git.
- Для `EXCEL-005` проверить внешние ссылки workbook на `[1]СП 20` и `[1]СП РК EN`.
- Для `EXCEL-005` отдельно разобрать лист `типы окон` и связь с коэффициентами приведения момента.
- Для `EXCEL-010` сопоставить `Сводка` с текущим `DEFAULT_COLUMN_INPUT`.
- Для `EXCEL-010` проверить типы колонн `крайняя`, `средняя`, `фахверковая` и влияние уклона кровли на длины колонн.
- Для `EXCEL-010` проверить формирование `N/M` и крановые нагрузки.
- Для `EXCEL-008` проверить, есть ли в workbook расчёт горизонтальной реакции `H`.
- Для `EXCEL-008` расшифровать `ОРб`, `ОР`, `РР` и связать их с текущими терминами UI.
- Для `EXCEL-009` проверить методику axial wind load для прокатных прогонов.
- Для `EXCEL-009` выбрать минимальные acceptance scenarios для прокатных прогонов и ЛСТК МП350/МП390.
- Для `EXCEL-007` сопоставить текущий модуль Beam Cell с workbook-терминами `ВБ`, `ГБ`, `К`.
- Для `EXCEL-007` выбрать acceptance scenario по ВБ/ГБ/массе/стоимости.
- Для `EXCEL-006` сопоставить UI подкрановой балки с входами `Сводка`.
- Для `EXCEL-006` выбрать acceptance scenario для одного и двух кранов.
- Для `EXCEL-006` проверять, что generated engine остаётся lazy-loaded и не возвращается в стартовый bundle.
- Для `KM-002` и `KM-003` визуально проверить страницы, где текст PDF извлекается как `(cid:...)`.
- Для `KM-002` и `KM-003` извлечь обезличенные геометрические сценарии для будущего раздела единого здания.

- Для `SCN-COLUMN-001` проверить, как обрабатывать крановые поля с `#N/A`.

## Что проверить перед следующим крупным refactor

Сверить документацию с фактическим кодом, прогнать typecheck/lint/test/build, проверить стартовый bundle и GitHub Pages.
